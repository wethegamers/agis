The `agis-bot` is technically mature in its infrastructure (Kubernetes/Agones/Vault integration is excellent) but **operationally immature** in its user-facing economy features.

You have successfully built the *engine* (backend services), but you are missing the *steering wheel* (user commands) for your most critical differentiator: the Guild Economy.

### üî¥ Critical Gaps (Must-Fix Before Launch)

These are features that exist in your `internal/services` (backend logic) but have **zero exposure** to the end-user via Discord commands. If you launch now, these features effectively do not exist for your users.

#### 1. The Guild Interface "Air Gap"
Your business model relies on Guilds to sell high-margin Titan servers.
* **Backend Reality:** `internal/services/guild_treasury.go` has full logic for `CreateGuild`, `DepositToGuild`, `AddMember`.
* **Frontend Reality:** `internal/bot/commands/guild_server_command.go` ONLY has commands to *spend* treasury money (`create`, `terminate`) and view it (`treasury`).
* **THE GAP:** Users have **no way to create a guild, invite members, or deposit money**. You have a vending machine with no coin slot.
* **Missing Commands needed immediately:**
    * `/guild create <name>`
    * `/guild invite <@user>`
    * `/guild join`
    * `/guild deposit <amount>`

#### 2. The Ad Dashboard "MVP Trap"
* **Current State:** `internal/http/server.go` (lines 798-846) serves a raw, unstyled HTML string as your ad dashboard.
* **Business Risk:** This is your primary revenue engine for 85% of users. Presenting a 1995-style HTML page will severely hurt trust and conversion rates. It needs, at minimum, a styled template matching your brand.

#### 3. Missing "Premium" Role Sync
* **Current State:** `internal/services/role_sync.go` only syncs one role type: `VERIFIED_ROLE_ID`.
* **Business Risk:** When a user subscribes via Stripe, they get premium *benefits* in the database, but they don't automatically get a "Premium Member" role in Discord to flaunt their status (a key psychological driver for subscriptions).
* **Fix:** Update `RoleSyncService` to also sync a designated `PREMIUM_ROLE_ID` based on `users.tier`.

### üîÆ Industry Standard "Missing Criticals"

If you want to compete with established hosts (even premium ones), you are missing these standard API/Bot features:

#### 1. RCON / Console Access (High Importance)
* **Standard:** Every game host allows users to see the live server console and send commands (e.g., `/op user`, `/gamemode creative`).
* **Current Status:** Missing. Users have no way to manage their server internals without you building a custom command for every possible game action.
* **Recommendation:** Implement a secure websocket or streaming API endpoint that proxies to the Agones SDK `ExecuteCommand` or attaches to the pod's stdin/stdout.

#### 2. Real-time Server Metrics for Users
* **Standard:** Users want to see "Why is my server lagging?" (CPU/RAM graphs).
* **Current Status:** You have these metrics internally for *yourself* via Prometheus, but no easy way for a *user* to see them via bot or API.
* **Recommendation:** A `/server stats <name>` command that pulls live CPU/Memory usage from the pod metrics.

#### 3. Automated Mod/Plugin Installer
* **Standard:** One-click install for popular plugins (PaperMC, Oxide for Rust).
* **Current Status:** Users likely have to manually FTP files (if FTP is even exposed? I don't see explicit FTP services in the charts).
* **Recommendation:** This is complex but necessary for retention long-term.
